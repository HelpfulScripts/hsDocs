  61## Creating the Documentation DocSet
  62Follow the instruction for `typedoc` in commenting the code and set the `json` option to create a 
  63json file containing the documentation. See the Configuration section below for details.
  64
  65In addition to the documented source files, `hsDocs` recognizes a special **`overview.ts`** file 
  66that will be displayed as a project overview. 
  67For this to work, `typedoc` requires the file to have two separate comment entries.
  68If the second comment is missing, `typedoc` will not generate a comment in the DocSet.
  69
  70DocSets are shown as menu tabs across the top of the window. To include a DocSet in the menu, ensure that the 
  71corresponding documentation `.json` file is copied into the `data` folder inside the `hsDocs` staging location 
  72for the http server. Then edit the `index.json` file to include the documentation file name in the "docs" section. 
  73
  74### Modules
  75Each source file is considered to be a `module`. Modules are shown in the left-hand overview panel of the main `hsDocs` window.
  76Alternatively, a `@module` statement in the summary comment at the top of a file will create, or add to, a module of the specified name.
  77This encourages to separate code into files without cluttering the overview.
  78
  79### Linking across the docsets
  80Links to items across the docsets can be placed via a link directive: 
  81- "{@link [*docset*:]*module*.*item* linked text}", 
  82
  83where
  84 - *docset* optional; the name of the docset. If omitted, a link within the current docSet will be created
  85 - *module* is name of the module to link to; 
  86   if *module* is `overview`, or unspecified, the link will point to the docset overview page.
  87 - *item* is the name, or period-separated path, of the item within the module. 
  88 - *linked text* is displayed with a link to path within module
  89
  90Examples:
  91- '{@link overview this hsDocs overview}' -> {@link overview this hsDocs overview}
  92- '{@link hsDocs: this hsDocs overview}' -> {@link hsDocs: this hsDocs overview}
  93- '{@link hsDocs:DocSets.DocSets.add the `add` function}' --> {@link hsDocs:DocSets.DocSets.add the `add` function}
  94- '{@link DocSets.DocSets.add the `add` function}' --> {@link DocSets.DocSets.add the `add` function}

 129## Configuring typedoc
